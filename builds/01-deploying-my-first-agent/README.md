# 🚀 Build 01: Deploying My First AI Agent

## 🎯 Objective

The objective of this build was to deploy Azure AI Foundry resources, deploy an AI model, and make my first secure AI agent call using Python.

This build helped me understand how a Python application communicates with a cloud-hosted Large Language Model (LLM) using Azure AI services.

---

## 🛠️ Technologies Used

* Azure AI Foundry
* Azure OpenAI
* Python
* Azure CLI
* Azure Identity
* OpenAI SDK
* Bicep (Infrastructure as Code)
* VS Code
* Git & GitHub

---

## 🏗️ Workflow

1. Created a Python virtual environment.
2. Installed project dependencies using `requirements.txt`.
3. Configured VS Code debugging using `launch.json`.
4. Installed and authenticated Azure CLI.
5. Created an Azure Resource Group.
6. Deployed Azure AI Foundry resources using a Bicep file.
7. Deployed a GPT-4.1-mini model.
8. Configured environment variables.
9. Executed my first AI agent call using Python.
10. Documented and published the build to GitHub.

---

## 📚 Key Concepts Learned

### Azure AI Foundry

A centralized platform to build, deploy, and manage AI applications.

### Azure AI Project

A workspace inside AI Foundry where AI applications are developed.

### Bicep

Infrastructure as Code (IaC) used to provision Azure resources.

### Azure Identity

Provides secure authentication without hardcoding API keys.

### Bearer Tokens

Temporary access tokens used to securely authenticate requests.

### Environment Variables

Store configuration values outside the source code.

### AI Agent Call

A structured request sent to an AI model consisting of system instructions and user messages.

## 🎯 Outcome

Successfully deployed Azure AI resources and executed my first secure AI agent call using Python.
